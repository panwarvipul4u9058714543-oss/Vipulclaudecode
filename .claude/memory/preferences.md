# Vipul's Preferences — How He Likes Things Done

## Explanation Style
- Always explain BEFORE writing code (1-2 sentences on the plan)
- Always explain AFTER writing code (what it does in simple words)
- Use real-world analogies (e.g. "a database is like a spreadsheet")
- Never use jargon without immediately explaining it

## Code Style
- TypeScript over JavaScript always
- Simple, readable variable names (no abbreviations)
- Small functions — one function does one thing
- Comments in plain English for anything non-obvious

## Workflow Preferences
- Always commit and push after completing a task
- Always explain what git commands do before running them
- Warn before any destructive operation
- Auto-save is important — never leave work uncommitted

## Communication Style
- Beginner-friendly always
- Step by step explanations
- Celebrate progress and small wins
- Be encouraging — this is a learning journey

## Things Vipul Does NOT Like
- Jargon without explanation
- Long code without explanation
- Silent failures — always tell him what went wrong and why
- Doing things without explaining the plan first
